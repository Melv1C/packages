@melv1c/ui-core provides a root provider component that handles global configuration for your app.

## UICoreProvider

`UICoreProvider` wraps your app with the necessary context providers for i18n (via `I18nextProvider`) and toast notifications (via `Sonner`). **You must wrap your application with this provider** — all @melv1c/ui-core components expect to be rendered inside it.

```tsx title="app.tsx"
import { UICoreProvider } from '@melv1c/ui-core';

export function App({ children }: { children: React.ReactNode }) {
  return (
    <UICoreProvider i18nConfig={{ locale: 'en' }} hasToaster>
      {children}
    </UICoreProvider>
  );
}
```

## Props

| Prop               | Type                   | Default     | Description                                                      |
| ------------------ | ---------------------- | ----------- | ---------------------------------------------------------------- |
| `i18nConfig`       | `string \| I18nConfig` | `'en'`      | Language configuration. Pass a locale string or a config object. |
| `hasToaster`       | `boolean`              | `false`     | Whether to render the Sonner toast provider.                     |
| `loadingComponent` | `React.ReactNode`      | `undefined` | Component to show while the i18n instance is loading.            |
| `children`         | `React.ReactNode`      | —           | Your application content.                                        |

## i18nConfig Options

### Simple locale string

```tsx
<UICoreProvider i18nConfig={{ locale: 'fr' }}>{children}</UICoreProvider>
```

### Config object with overrides

```tsx
<UICoreProvider
  i18nConfig={{
    locale: 'en',
    overrides: {
      loginForm: {
        title: 'Welcome',
      },
    },
  }}
>
  {children}
</UICoreProvider>
```

### Bring your own i18n instance

```tsx
<UICoreProvider i18nConfig={{ instance: myI18nInstance }}>{children}</UICoreProvider>
```

## Toaster Configuration

When `hasToaster` is `true`, the provider renders a `<Toaster />` component from [Sonner](https://sonner.emilkowal.dev/). You can then trigger toasts anywhere in your app:

```tsx
import { toast } from 'sonner';

function MyComponent() {
  return <Button onClick={() => toast.success('Settings saved!')}>Save</Button>;
}
```

## Loading State

If your i18n configuration requires async loading (e.g., fetching translations from a server), you can provide a loading component:

```tsx
<UICoreProvider i18nConfig={{ locale: 'en' }} loadingComponent={<div>Loading translations...</div>}>
  {children}
</UICoreProvider>
```

## Provider is Required

All @melv1c/ui-core components should be rendered inside `UICoreProvider`. Using components without the provider is not supported and may lead to unexpected behavior.
